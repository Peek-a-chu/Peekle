## 1. 프로젝트 원칙 (매우 중요)

### 1.1 기술 스택 및 데이터 처리

- **Frontend:** Next.js (App Router), TypeScript
- **Backend (Real-time & Media):** Spring Boot
    - **WebSocket:** SockJS (실시간 채팅, 게임 상태 동기화)
    - **WebRTC:** LiveKit (화상 채팅, 화면 공유)
    - **API Communication:** `fetch` / `axios`를 통한 백엔드 통신 허용
- **External Links:** BOJ(백준) 등 외부 이동이 필요한 경우, `target="_blank"`를 사용하여 새 탭으로만 허용

### 1.2 UI/UX 표준

- **Routing:** Next.js **App Router** 기반의 실제 라우팅 구현
    - 페이지 별 route를 만들어서 네비게이션이 동작해야 함
- **Theme:** 전역 **화이트/다크 모드** 지원 + **메인 테마 컬러** 선택 토글 기능 제공
    - 테마는 localStorage로 유지
- **Accessibility:** 시맨틱 태그 사용, 키보드 포커싱, ARIA 속성 등 웹 접근성 베스트 프랙티스 준수

## 2. 라우팅 (Page Structure)

- `/` 랜딩 페이지
- `/login` 로그인
- `/signup` 회원가입
- `/home` 메인 대시보드
- `/study` 스터디 방 목록
- `/study/[id]` 스터디 방
- `/game` 게임 방 목록
- `/game/new` 게임 방 생성
- `/game/[id]` 게임 방 (대기/진행 상태 토글로 목업)
- `/workbook` 문제집 목록
- `/workbook/new` 문제집 만들기 (목업 폼)
- `/search` 통합 검색
- `/profile/[id]` 유저 정보 (프로필)
- `/ranking` 랭킹
- `/league` 리그 정보

## 3. 전역 레이아웃 및 UI 구성 (App Shell)

### 3.1 좌측 네비게이션바 (고정)

- 위치: 화면 왼쪽 고정 (스크롤 무관)
- 예외 처리: `/study/[id]`, `/game/[id]` 진입 시 제거되며, 대신 좌측 상단에 `뒤로 가기` 버튼 제공
- 구성 요소:
    1. 프로필 영역 (최상단):
        1. 프로필 이미지 + 닉네임 표시
    2. 내 리그 정보 카드 (프로필 직하단):
        1. 구성: 리그 아이콘, 등급명(예: 골드 클러스터), 점수, 순위/전체 인원, 상태 배지(승급예정/유지/강등위기)

    3. 메뉴 리스트 (상→하):
        1. 메인 (`/home`)
        2. 스터디 방 (`/study`)
        3. 게임 방 (`/game`)
        4. 문제집 (`/workbook`)
        5. 랭킹 (`ranking`)
        6. 리그 (`/league`) 
        7. 검색 (`/search`)
    4. 설정 버튼 (최하단): 클릭 시 [3.2 설정 모달] 호출

### **3.2 전역 설정 모달**

- 좌측 하단 ‘설정’ 버튼 클릭 시 등장

    - 카메라 미리보기 기능
    - 볼륨 조절 슬라이더 및 테스트 기능

### 3.3 상단 프로필 드롭다운

- 프로필 영역 클릭 시, 드롭다운으로 아래 메뉴 노출

    - 로그아웃:
        - Mock 데이터(로그인 상태) 초기화 후 `/login` 페이지로 이동

## 4. 공통 모달/정책

### 4.1 룸 입장 전 프리조인 모달 (스터디/게임 공통)

- 트리거: `/study/[id]`, `/game/[id]` 페이지 진입 시 “입장 전 프리조인 모달”을 띄우는 흐름을 제공
- UI 구성:
    - 모달 내부에 스크롤 제공
    - 미디어 미리보기: 카메라/마이크 미리보기(실제 미디어 연결 X → Placeholder 카드로 대체)
    - 마이크/스피커 장치설정 및 볼륨 조절, 테스트
    - 장치 설정: 마이크/스피커 선택 드롭다운 (Mock List)
    - 볼륨 조절: 입력 레벨 슬라이더 (0~100% 드래그 가능)
    - 테스트:
        - 마이크에 실제로 입력이 들어가는지 음성 크기가 보이게
        - 스피커엔 임의로 소리 재생. 볼륨조절할경우 실시간으로 변환됨
    - 확장 프로그램 체크:
        - 설치됨: `입장하기` 버튼 활성화
        - 미설치: `입장하기` 버튼 비활성화
            - 대신 확장프로그램 설치 안내가 나와있는 프로필 페이지로 이동하는 버튼 제공

### 4.2 초대하기 모달

- 트리거: 룸 내부 상단 “초대하기” 버튼 클릭 시 등장
- 기능:
    - 현재 방 링크 표시
    - [복사] 버튼 (클립보드 복사 Mock 액션)

### 4.3 확장프로그램 공통 안내

- 로그인 시 로직:
    - “확장프로그램 미설치 상태”로 로그인하면 **로그인마다 안내 모달** 노출(닫기 가능)
    - 모달 내 버튼: [설치 안내 보러가기] → `/profile/[id]` 로 이동 (이곳에 설치 안내가 나와있어서)
- 상태 정의 (3가지 상태, localStorage로 관리):
    - (1) 미설치
    - 설치됨
        - (2) 미연동
        - (3) 연동 완료
    - 프로필 페이지 내에 토글 스위치를 두어, 확장 프로그램 설치 상태를 쉽게 바꿀 수 있게 제공 (데모용)

### 5.7 스터디 방(`/study/[id]`)

- **상단 바**
    - 좌측:
        - [뒤로가기] 버튼
        - 방 제목
    - 우측:
        - [초대하기] 버튼
            - 클릭 시 초대 모달(링크 복사 등)
        - […] 더보기 아이콘
            - if) 방장 → 방 설정창: 제목 변경, 설명 변경, 삭제 버튼
            - if) 참여자 →  스터디 탈퇴 버튼(모달로 다시 한 번 묻기)
- **좌측 사이드바**
    - 접을 수 있음
    - 상단:
        - [달력] (날짜 필터)
        - [문제 추가] 버튼 (참여자 누구나 가능)
    - 문제 목록 (선택된 날짜):
        - 기본 상태: 난이도/태그 숨김
        - 항목 구성:
            - 전구(힌트) 토글:
                - 클릭 시 해당 문제의 난이도/태그 표시
            - 문제:
                - 문제 번호 / 제목 + [BOJ 이동] 버튼(새 탭)
            - 제출 현황: 문제마다 제출 완료 인원수 `3/10` 표시 (목업 집계)
            - 풀었는지 여부: 기본적으로 사용자가 풀었는지 여부를 표기
                - 남의 화면을 볼때는 추가로 해당 유저가 풀었는지 여부를 표기
        - 관찰 상태 표시:
            - 여러 문제 중에서, 어떤 문제에 대한 코드를 확인하는 것인지 UI상 즉시 확인 가능하도록 하이라이트 표시
                - 내 코드를 볼 때도
                - 다른 사람의 코드를 볼 때도
                - → 어떤 문제에 대한 열람인지 명확히 하이라이트 주기
- **중앙 영역 (화상 + IDE + 카메라/마이크/화이트보드)**
    - **상단: 화상 타일**
        - 접을 수 있음
        - 표시: 카메라 OFF 시, 프로필 이미지 카드로 대체
        - 정렬: 타일 정렬 규칙을 UI로 반영(화이트보드가 열리면 1번 타일에 썸네일)
            - 정렬 기준:
                - 항상 첫번째는 본인의 캠 타일이 위치한다.
                    - 단, 하단의 화이트보드 버튼을 눌러서 화이트보드가 활성화 되었을 시, 화이트보드는 내 캠보다 먼저 위치한다.
                - 누군가 마이크를 키고 말하면, 본인의 캠 다음에 우선 위치한다.
        - 관찰 상태 표시:
            - if) a의 코드를 b,c,d가 보고 있으면 a의 타일 우측하단에 b,c,d의 프로필아이콘이 나오게
                - 영역이 넓지 않으므로, 일부만 보여주고 나머지는 +6 이런식으로 인원수를 보여준다.
                    - hover 또는 클릭하면 프로필 목록을 볼 수 있다.
    - **중단: IDE (Code Editor)**
        - 레이아웃: VSCode 유사 스타일 (목업)
        - 기능: 언어 선택(C++/Java/Python), 코드 복사, 색상 변경(다크/화이트모드로 ide 색상만 변경할 수 있게)
        - 제출 프로세스 (보조 플로우):
            1. [제출] 버튼 클릭 
            2. `navigator.clipboard.writeText()`로 코드 자동 복사 
            3. BOJ 제출 페이지 새 탭 오픈 (문제 번호 기반 URL)
            4. (가능하면) BOJ 화면에서 언어 변경 URL 파라미터 적용 
            5. 안내 문구: 최종 제출은 사용자 직접 수행이라는 안내 문구 표시 
        - 관찰 상태 표시:
            - if) 다른 참여자의 타일 클릭 (다른 사람 코드 보드를 볼 수 있음)
                - → IDE가 해당 유저 코드로 전환되고 **읽기 전용 모드** (당연히 읽기 전용이고, 읽기 전용이라는 표기는 굳이 보여줄 필요 없음)
            - if) 내가 남의 코드를 보고 있으면
                - “타인의 코드 열람 중”이 확실히 보이도록 내 IDE화면 에 타인의 코드 열람중 이 뜨고 테두리 색 변경
        - 제출 상태 표시:
            - 제출이 완료된 코드인지 아닌지 구분이 가능해야 함
        - 칭찬 기능: 다른 사람의 코드에 대해 이모지 남길 수 있음. 타인이 내 코드에 이모지를 남기면, 나는 그만큼 리그에서 사용하는 점수가 오름.
            - 👍🏼 3점 - 잘 짠 코드군
            - 🆗 2점 - 확인
            - 🤔 1점 - 흠…
        - 내 코드로 복귀:
            - 내 타일 클릭 시 내 코드로 돌아오기
    - **하단:**
        - **카메라/마이크/화이트보드:**
            - 카메라, 마이크 on/off 가능
            - 화이트보드 버튼: 클릭하면 화이트보드가 열리고 상단의 화상 타일에도 등장
            - 카메라/마이크/스피커 설정을 할 수 있는 모달을 등장시키는 버튼
- **우측 패널(채팅/참여자 탭)**
    - 탭: 채팅 / 참여자 목록
    - 접기(닫기) 가능
    - 채팅
        - 마크다운 렌더(목업 메시지로만)
    - 참여자
        - 방장의 권한
            - 본인의 아이콘 옆에는 모두 음소거 버튼이 생김
            - 참여자의 마이크/카메라 오프 가능(온은 불가능)
            - 참여자 강퇴
            - 방장 권한 넘기기 버튼(경고가 나와야함)
            - 스터디 종료 버튼
- 화이트보드
    - 하단에 “화이트보드” 버튼 제공(실제 화이트보드 X → placeholder)
    - 열리면 화상 타일 목록에 “화이트보드 썸네일 타일”이 1번으로 나타나게

### 5.8 게임 목록(`/game`)

- 상단: 타이틀 + “게임방 생성하기”(`/game/new`)
- 게임 종류 4개
    - 카드에 게임에 대한 설명 간단히 나와있음
    - 개인전 (시간제한 많이 풀기/스피드 레이스)
    - 팀전 (시간제한 많이 풀기/스피드 레이스)
- 먼저 개인전, 팀전 여부 스위치 토글로 선택 하기
    - 개인전: 빨간색
    - 팀전: 파란색
- 2개의 게임(시간제한 내 많이 풀기 / 스피드 레이스) 카드 선택
    - 시간제한 내 많이 풀기: 제한시간 내 문제 많이 풀기(10분~4시간)
    - 스피드 레이스: 1개만 빨리 풀기 - 만장일치 스킵가능
- 해당 게임 유형을 누르면 해당하는 게임방 목록이 뜸
- 게임방 목록 카드 + 공개/비공개 + 입장 버튼(비공개는 비번 모달 목업)
- 검색 바

### 5.9 게임방 생성

- 모달으로 만들기
- 방 제목
- 게임 모드(4종)  중 선택 : 게임방 목록에서의 ui와 같게

    - 해당 모달 안에서 출제 방식 및 문제 목록 설정 가능하게
    - A) BOJ 랜덤(리그/태그 기반): 난이도 범위 슬라이더 + 최소/최대 리그 드롭다운 + 태그 선택
    - B) 문제집 랜덤: 문제집 선택 + 문제 리스트 + 제거 + 검색으로 추가(전부 목업)

### 5.13 프로필(`/profile/[id]`)

- 상단 프로필 카드: 사진/닉네임/팔로우/스트릭/리그 등급·점수/푼 문제 수
- 사진/닉네임 클릭 시 사진/닉네임 변경하는 모달등장
- 모달내에서 닉네임을 입력할때마다 중복체크
- 팔로우/팔로워 버튼 및 클릭 시 팔로우/팔로워 목록이 나옴
    - 목록에서 내가 아는 사람이 나오면 표기
- **그 아래엔 스트릭(잔디) + 일별 풀이 기록**
    - 스트릭 화면 구성
        - 상단 오른쪽: 연도 드롭다운
        - 스트릭 바로 아래: 월 단위 구분 표시
        - 왼쪽: 요일 표시 (일~토)
    - 기본 선택 날짜: 오늘
    - 날짜 클릭 시 하단 "일별 풀이 기록" 갱신
        - 풀이 기록이 없는 경우
            - 빈 상태 문구 표시
            - 구분: 오늘 / 과거
        - 풀이 기록 표시
            - 각 문제별 표시
                - 문제 누르면 백준 사이트로 바로 가기
            - 문제 위치 표시
                - 스터디에서 푼 문제: `스터디 방 이름` 라벨 표시
                - 게임에서 푼 문제:
                    - 게임 유형 라벨 표시 (`팀/개인`, `게임 이름`)
                    - `게임방 이름` 라벨 표시
                - 그냥 푼 문제:
                    - 별도의 라벨 표시 없음
- 푼 문제 히스토리(/profile/[id]/history)
    - 내가 푼 문제를
    - 시간순서대로 페이지네이션
    - 필터
        - 언제 풀었는지 기간 언제부터 언제
        - 티어
        - 어디에서 풀었는지(스터디or게임or그냥풀기)
    - 푼 날짜, 메모리, 시간, 언어, 문제이름, 성공/실패 여부
    - 해당 문제 클릭할 경우(본인만 가능) 오른쪽에 슉하고 나오게 하기
        - 코드, 푼 날짜, 메모리, 시간, 언어, 문제이름, 성공/실패 여부 가 나오고 복사 버튼 존재
- 본인일 때:
    - 프로필 사진 변경 모달(목업)
    - 닉네임 실시간 중복 체크  —- 닉네임 변경 시 말이죠?
    - **확장프로그램 섹션(명확한 블록)**:
        - 상태 표시(미설치/설치됨-미연동/연동완료)
        - 설명 모달
        - 설치 버튼(외부 링크 새 탭 — 임시 URL)
        - 토큰 발급/복사(목업 토큰 생성 후 클립보드 복사)
    - 탈퇴 버튼
