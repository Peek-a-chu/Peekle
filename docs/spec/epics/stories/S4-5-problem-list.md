## 🧾 User Story

**As a** 사용자
**I want to** 날짜별로 배정된 문제를 확인하고 관리하고 싶다
**So that** 스터디 일정에 맞춰 체계적으로 문제를 풀기 위함이다

---

## ✅ Acceptance Criteria

1. **캘린더 연동**
   - 달력에서 날짜 선택 시 해당 일자에 추가된 문제 목록이 표시되어야 한다.

2. **문제 관리**
   - 문제 추가 시 이름/번호로 검색하여 등록할 수 있어야 한다.

3. **문제 표시**
   - 각 문제는 '제목', '풀이 현황(푼 인원)', '내 풀이 여부(Check)'가 표시되어야 한다. (전체 인원 분모 제거)

4. **힌트 및 공유**
   - [힌트/남의 코드 보기] 버튼을 통해 티어 힌트를 보거나 제출된 동료들의 코드를 열람할 수 있어야 한다.

### 문제 카드 상세

| 요소                 | 설명                                               |
| -------------------- | -------------------------------------------------- |
| **🔗 외부 링크**     | 클릭 시 백준/SWEA 문제 페이지로 이동 (새 탭)       |
| **💡 힌트 토글**     | 클릭 시 티어(난이도)와 태그 힌트 표시/숨김         |
| **[👁] 보기 아이콘** | 클릭 시 풀이 보관함 모달 열기                      |
| **✓ 풀이 완료**      | 내가 해당 문제를 성공적으로 제출한 경우 표시       |
| **👤 N/M**           | 푼 인원 수 / 전체 스터디 인원 수                   |
| **태그 칩**          | `[bfs]`, `[다이나믹 프로그래밍]` 등 알고리즘 분류  |
| **선택 상태**        | 현재 IDE에서 작업 중인 문제는 분홍색 배경으로 강조 |
